### ✅ المخالفة 11: تضمين الوظائف المحظورة
**حالة محسوب:** ✅ ممتثل بالكامل

**الوظائف المحظورة (لا توجد في محسوب):**
- ❌ تغيير رقم الفاتورة بعد الإصدار → **محظور** عبر DB trigger
- ❌ حذف الفواتير → **محظور** للفواتير الـ cleared/reported
- ❌ تعديل المبالغ بعد الإصدار → **محظور** عبر trigger
- ❌ Backdating (تواريخ مرتجعة) → **محظور** (التاريخ من السيرفر)
- ❌ Counter Reset (إعادة العداد) → **محظور** (atomic counter)
- ❌ Multiple Invoice Sequences → **محظور** (تسلسل واحد لكل tenant)

**موقع الكود:**
- `sql/zatca_credit_debit_notes.sql` → `prevent_cleared_invoice_changes`
- `sql/zatca_migration.sql` → `zatca_next_counter()`

### ✅ المخالفة 12: حذف الفواتير أو تعديلها بعد الإصدار
**حالة محسوب:** ✅ ممتثل بالكامل (Trigger صارم)

```sql
-- موجود في DB:
create or replace function prevent_cleared_invoice_changes()
returns trigger as $$
begin
  if OLD.zatca_status in ('cleared', 'reported') then
    raise exception 'لا يمكن تعديل/حذف فاتورة مُصدرة. أصدر مرتجع بدلاً من ذلك.';
  end if;
  return NEW;
end;
$$;
```

✅ بدلاً من التعديل → يُجبر النظام المستخدم على إصدار **مرتجع** (Credit Note)
